# positioning-book-skill｜《定位》AI Skill 蒸馏包

> 把《定位》里的营销判断方法，变成可以直接调用的 AI 工作流。  
> 它不是读书笔记，而是一套用于**产品定位、品牌表达、广告文案、命名判断、竞品分析**的实用工具包。

---

## 这是什么？

这是一个基于《定位》（Al Ries & Jack Trout）蒸馏出来的 AI Skill Pack。

它的目标不是复述书中内容，而是把书里的核心方法论拆成一组可以反复调用的技能，让 AI 在真实工作中帮你做判断，比如：

- 一个产品到底应该主打什么？
- 文案是不是太散、太满、太像自嗨？
- 一个品牌名是否容易进入客户心智？
- 新产品要不要沿用老品牌名？
- 弱势品牌如何避开头部品牌，找到一个能占住的位置？
- 做公众号、海报、官网首屏、销售话术时，应该保留什么、舍弃什么？

一句话：

> 这套 skill 不是帮你“多写一点”，而是帮你“少说一点，说准一点”。

---

## 适合谁用？

### 1. 市场 / 新媒体 / 公众号运营

适合用来判断一篇文章的切入角度：

- 文章标题是否有明确心智钩子
- 开头是否能快速建立“这篇文章和我有关”
- 内容是否围绕一个定位点展开，而不是卖点堆叠
- 公众号选题是否能避开同质化表达

### 2. 平面设计 / 产品海报 / 宣传物料

适合用来检查海报文案和画面主信息：

- 主标题是否过于抽象
- 一张海报是不是塞了太多卖点
- 客户 3 秒内能不能看懂“你是谁、你解决什么问题”
- 视觉中心和传播中心是否一致

### 3. 产品专员 / 产品经理

适合用来做产品定位和功能取舍：

- 新产品应该进入哪个品类阶梯
- 是否有机会成为某个细分场景的“第一”
- 产品资料应该突出哪一个核心差异
- 哪些功能只是内部优势，不适合作为外部定位

### 4. 销售 / 渠道 / 海外业务

适合用来整理销售话术：

- 面对竞品时，如何避免硬碰硬
- 如何用客户已有认知解释自己的产品
- 如何给经销商一个更容易转述的定位句
- 如何判断“低价”“国产替代”“服务响应”等说法是否足够锋利

### 5. 创始人 / 管理层 / 品牌负责人

适合用来做战略层面的判断：

- 公司到底要占哪个心智位置
- 品牌延伸是否会稀释原有认知
- 是否应该做子品牌或新品牌
- 公司宣传是否长期保持一致

## 这个 skill 包含哪些能力？

当前拆成 10 个可调用 skill：

| Skill | 用途 |
|---|---|
| `outside-in-position-diagnosis` | 从客户心智出发诊断定位，而不是从产品自嗨出发 |
| `simple-message-compression` | 把复杂卖点压缩成一个清晰、易记的信息 |
| `first-in-mind-category-entry` | 判断能否成为某个新品类/新场景的“第一” |
| `mental-ladder-mapping` | 画出客户脑中的品牌阶梯和竞争位置 |
| `leader-defense-positioning` | 帮领导品牌防守已有位置 |
| `follower-gap-positioning` | 帮跟随者寻找空位，避开正面竞争 |
| `reposition-competitor` | 通过重新定义竞争对手，创造自己的位置 |
| `naming-hook-design` | 判断品牌名、产品名、功能名是否有心智钩子 |
| `product-extension-kill-switch` | 判断产品延伸是否会伤害原品牌 |
| `six-step-positioning-project` | 用六步法完整推进一次定位项目 |

## 怎么用？

### 在 ChatGPT / Claude / Gemini 中使用

把整个仓库或 `books/positioning-ries-trout/` 文件夹上传给 AI，然后这样说：

```text
请先读取 books/positioning-ries-trout/INDEX.md，判断我的任务应该调用哪些《定位》skill。
不要直接写文案，先做定位判断，再输出结果。
```

### 在 Claude Code / Codex 中使用

把仓库放到项目目录，然后对 agent 说：

```text
这是一个《定位》AI skill pack。处理营销、产品定位、命名、海报文案、官网文案、竞品分析任务时，请优先读取 books/positioning-ries-trout/INDEX.md，再按任务选择对应 SKILL.md。
```

## 官方推荐提示词

下面这些 prompt 可以直接复制使用。更多场景见 [`OFFICIAL_PROMPTS.md`](./OFFICIAL_PROMPTS.md)。

### 产品定位

```text
请读取 books/positioning-ries-trout/INDEX.md，调用《定位》相关 skill，帮我分析这个产品应该如何定位。

请按以下顺序输出：
1. 当前客户心智中已有的品类/品牌阶梯；
2. 我们现在是领导者、跟随者还是新进入者；
3. 可以争夺的心智空位；
4. 应该舍弃的卖点；
5. 应该保留并反复强化的核心信息；
6. 一句话定位表达；
7. 这个定位不适用的边界。

不要写成普通宣传文案，要先做判断。
```

### 公众号文章选题

```text
请用《定位》skill 帮我优化这篇公众号文章选题。

目标读者：{填写读者，例如 ICU 护士长 / 经销商 / 产品经理 / 医生}
原始选题：{填写选题}
核心产品或主题：{填写产品/主题}

请输出：
1. 读者脑中已有认知；
2. 这个选题现在的问题；
3. 更容易进入心智的切入角度；
4. 3 个标题方向；
5. 最推荐的标题；
6. 开头第一段应该怎么写。
```

### 产品海报文案

```text
请调用 simple-message-compression、outside-in-position-diagnosis 和 naming-hook-design，帮我优化这张产品海报文案。

产品：{产品名称}
目标客户：{客户群体}
原始卖点：{卖点列表}
使用场景：{展会 / 科室拜访 / 官网 / 朋友圈 / 经销商物料}

请输出：
1. 哪些卖点应该删除；
2. 哪个卖点最适合作为主标题；
3. 主标题 5 个版本；
4. 副标题 3 个版本；
5. 海报上最多保留的 3 条辅助信息；
6. 不建议使用的空泛词。
```

## 判断输出好不好：一个简单标准

好的《定位》skill 输出，应该帮你做三件事：

1. **判断位置**：客户脑子里你现在在哪里？
2. **做出取舍**：哪些卖点必须删掉？
3. **形成钩子**：最后留下的一句话是否容易被客户记住？

不好的输出通常长这样：

- “高品质、高性能、高性价比、智能化、专业化”全都要；
- 只分析自己产品，不分析客户已有认知；
- 只写漂亮文案，不判断竞争位置；
- 不敢舍弃卖点。

## 重要边界

这套 skill 适合做营销定位、传播表达和产品认知判断，但不替代：

- 市场调研
- 临床证据
- 注册证与法规审查
- 招投标参数验证
- 医疗器械合规宣传审核
- 商标检索与法律意见

尤其在医疗器械场景中，所有竞品比较都必须基于可验证事实，避免虚假宣传或贬损式表达。
